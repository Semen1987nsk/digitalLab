## 🏗️ АРХИТЕКТУРНАЯ ФИЛОСОФИЯ (Суперсеньор-уровень)

### Принцип «Уровневой зрелости»

Вместо линейного Sprint 2 → 3 → 4 я предлагаю модель **уровней зрелости функций**, где каждая фича проходит 3 стадии:

```
L1 (MVP)       → Минимальная рабочая версия (1-2 дня)
L2 (Полезная)  → Покрывает 80% сценариев учителя (доп. 2-3 дня)
L3 (Мировой)   → На уровне Vernier/PASCO (доп. 3-5 дней)
```

**Почему это лучше**:
- L1 всех критических фич за 2 недели = можно показать учителю
- L2 по мере обратной связи = не строим лишнего
- L3 только для дифференциаторов = фокус ресурсов

### Принцип «Калибровочного конвейера»

Текущая калибровка — простой zero-offset для дальномера. Нужна **система калибровки** как у Releon/Vernier:

```
┌─────────────────────────────────────────────────────┐
│            CALIBRATION PIPELINE                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Raw ADC → [Factory Cal] → [User Cal] → [Unit]    │
│                 │               │           │       │
│           NVS/flash       SQLite/app    Converter  │
│                                                     │
│  Типы калибровки:                                  │
│  ├── Zero-offset (1 точка) ← уже есть для dist    │
│  ├── Two-point (2 точки, линейная)                 │
│  ├── Multi-point (3+ точек, сплайн)                │
│  └── Factory reset (возврат к заводским)           │
│                                                     │
│  Датчики с обязательной калибровкой:               │
│  ├── pH-метр (буферы 4.00, 7.00, 10.00)           │
│  ├── Вольтметр (опорное напряжение)                │
│  ├── Амперметр (шунт, offset)                      │
│  ├── Дальномер (zero + известное расстояние)       │
│  └── Термопара (ледяная вода 0°C + кипяток)        │
└─────────────────────────────────────────────────────┘
```

### Принцип «Графического конвейера» (Сопряжение графиков)

Текущая архитектура: 1 ExperimentPage → 1 SensorType → 1 график. 
Нужна **композитная модель** для сопряжения:

```
┌─────────────────────────────────────────────────────┐
│            GRAPH COMPOSITION ENGINE                 │
├─────────────────────────────────────────────────────┤
│                                                     │
│  GraphLayer (абстракция одного слоя данных):        │
│  ├── LiveDataLayer   — поток от активного датчика   │
│  ├── HistoryLayer    — данные из SQLite             │
│  ├── PredictionLayer — нарисованная гипотеза        │
│  ├── TheoryLayer     — теоретическая кривая f(x)    │
│  └── FitLayer        — результат аппроксимации      │

│  GraphComposer (управление наложением):             │
│  ├── addLayer(layer, axis: AxisSide.left)           │

│  ├── setXYMode(xLayer, yLayer)  // параметрический  │
│  ├── syncTimeRange(layer1, layer2)                  │
│  └── setDualAxis(leftLayers, rightLayers)           │
│                                                     │
│  7 сценариев совмещения:                            │
│  1. Мульти-канал (T+P+H с одного датчика)     ✅   │
│  2. Dual Y-axis (T °C слева, P Па справа)     ⭐   │
│  3. Overlay run (текущий + предыдущий)         ⭐   │
│  4. Cross-device (2 датчика → 1 график)        ⭐   │
│  5. X-Y parametric (V по X, A по Y → ВАХ)     ⭐   │
│  6. Theory overlay (y=mg + реальные данные)    ⭐   │
│  7. Prediction (нарисованная гипотеза)         ⭐   │
└─────────────────────────────────────────────────────┘
```

## 📐 КАЛИБРОВКА — Детальный дизайн

### Почему калибровка — P0 (обязательно)

1. **Нормативно**: Приказ №838 требует «калибровка датчиков в ПО»
2. **Конкуренты**: Releon имеет пошаговую калибровку. L-micro имеет
3. **Физика**: Без калибровки школьный вольтметр может показывать ±0.3В → ученик получает неверный закон Ома
4. **Текущее состояние**: Только zero-offset для 1 датчика (дальномер). Нет UI, нет persist, нет multi-point

### Архитектура калибровки (3 уровня)

```
┌─────────────────────────────────────────────────┐
│  УРОВЕНЬ 1: Заводская калибровка (firmware)     │
│  ─────────────────────────────────────────────   │
│  • Хранится в NVS ESP32                         │
│  • Устанавливается при производстве              │
│  • Не меняется пользователем                    │
│  • Уже частично есть (INA226 NVS)               │
├─────────────────────────────────────────────────┤
│  УРОВЕНЬ 2: Пользовательская калибровка (app)   │
│  ─────────────────────────────────────────────   │
│  • Хранится в SQLite (Drift) на устройстве      │
│  • Wizard в UI: 1-point / 2-point / 3-point     │
│  • Привязка к serial number датчика             │
│  • Factory reset = удалить профиль              │
├─────────────────────────────────────────────────┤
│  УРОВЕНЬ 3: Сессионная калибровка (RAM)         │
│  ─────────────────────────────────────────────   │
│  • Кнопка «Ноль» (zero-offset) ← уже есть      │
│  • Живёт только до конца эксперимента           │
│  • Быстрая, без мастера                         │
│  • Применяется ПОВЕРХ уровней 1+2               │
└─────────────────────────────────────────────────┘
```

## 📊 СОПРЯЖЕНИЕ ГРАФИКОВ — Детальный дизайн

### Почему graph overlay — стратегически важен

1. **Pedagogy**: Наложение позволяет ученику СРАВНИВАТЬ — это основа научного метода
2. **Конкуренты**: Vernier Pro + PASCO + GlobiLab имеют. Releon — НЕТ. → Мгновенное преимущество
3. **Физика**: ВАХ (X-Y), PV-диаграммы, сравнение нагрева/охлаждения — невозможны без overlay
4. **Prediction tool**: «Нарисуй гипотезу → проведи эксперимент → сравни» — величайшая педагогическая фича (есть только у PASCO!)

## 🧠 СУПЕРСЕНЬОР-ИНСАЙТЫ

### 1. «Prediction Tool — не фича, а философия»

PASCO выделяет Prediction как **основу pedagogy-driven design**. Когда ученик сначала рисует гипотезу, а потом видит реальность — это запускает cognitive conflict (когнитивный конфликт), который является самым эффективным механизмом обучения по Пиаже. **Ни один российский конкурент этого не имеет.** Это наш главный дифференциатор в категории «педагогические инновации».

### 2. «Калибровка — не про точность, а про доверие»

Школьный учитель физики видит 5.023 В вместо ожидаемых 5.000 В и теряет доверие к прибору. Калибровка решает не техническую, а **психологическую** проблему. Wizard должен быть простым как «нажми 3 кнопки», а не научным прибором.

### 3. «X-Y plot важнее dual Y-axis»

Dual Y-axis — визуальный сахар. X-Y plot — **фундаментальный инструмент физики**. ВАХ (вольт-амперная характеристика) — это первый X-Y график, который строит каждый ученик 8 класса. Без X-Y plot мы не можем проводить 30% стандартных лабораторных работ.

### 4. «GraphLayer — ключевая архитектурная абстракция»

Все 7 сценариев overlay сводятся к одной абстракции: `GraphLayer`. Если мы правильно спроектируем `GraphLayer` + `GraphComposer`, то Prediction, Theory overlay, History overlay, Cross-device — всё это станет **тривиальным** добавлением нового типа `Layer`. Инвестиция в архитектуру сэкономит месяцы.

### 5. «Не JSON — YAML для лабораторных»

JSON не поддерживает комментарии. Учитель, редактирующий лабораторную работу, хочет видеть пояснения. YAML + front matter (как в Hugo/Jekyll) = идеальный формат для лабораторных работ:

```yaml
# Лабораторная работа №1
title: Закон Ома для участка цепи
grade: 8
duration_min: 45
sensors: [voltage, current]
graph_mode: xy  # V по X, I по Y
expected_fit: linear  # y = kx + b → R = 1/k

steps:
  - instruction: Соберите цепь по схеме
    image: ohm_circuit.png
  - instruction: Установите напряжение 1 В
    action: record_point  # одиночный замер
  - instruction: Увеличивайте напряжение с шагом 1 В до 5 В
    action: record_series
  - instruction: Постройте ВАХ (V по X, I по Y)
    action: show_xy_graph
  - instruction: Найдите наклон прямой
    action: linear_fit
    expected_answer: 
      parameter: slope
      unit: А/В
      range: [0.08, 0.12]  # допустимый диапазон
```
