## 2. Технические рекомендации

### 2.1 Аппаратная часть

#### Микроконтроллер
| Параметр | Рекомендация | Обоснование |
|----------|--------------|-------------|
| MCU | **ESP32-S3-WROOM-1** | BLE 5.0, USB OTG, 2 ядра, PSRAM |
| Альтернатива | nRF52840 | Если нужен BLE Mesh |

#### Датчики для "Мультидатчик Физика"

| Параметр | Чип | Интерфейс | Цена ~$ |
|----------|-----|-----------|---------|
| V/A (напряжение/ток) | **INA226** | I²C | 2-3 |
| T/P/H (температура/давление/влажность) | **BME280** | I²C | 3-5 |
| Акселерометр/Гироскоп | **LSM6DS3** | I²C | 3-4 |
| Термопара | **MAX31855** | SPI | 4-6 |
| Расстояние ToF | **VL53L1X** | I²C | 8-12 |

#### Ключевые решения

1. **Гальваническая развязка (ADuM1201)**
   - Защита USB-порта ноутбука
   - Маркетинг: "Система защиты порта"

2. **PSRAM для буферизации**
   - 2-8 МБ внешней RAM
   - Режим "Осциллограф" с 100 кГц

3. **Физический выключатель питания**
   - Защита от глубокого разряда Li-Ion
   - Аккумуляторы живут 5+ лет

### 2.2 Программная часть

#### Стек технологий

| Компонент | Технология | Почему |
|-----------|------------|--------|
| Мобильное/Десктопное приложение | **Flutter 3.x (Dart)** | Одна кодовая база для всех ОС |
| State Management | **Riverpod** или **BLoC** | Строгое разделение |
| База данных | **Drift (SQLite)** | Оффлайн-режим |
| BLE | **flutter_blue_plus** | Стабильная работа |
| Графики | **FL Chart + LTTB** | Производительность |
| Прошивка | **ESP-IDF 5.x** | Полный контроль |
| BLE Stack | **NimBLE** | Легче Bluedroid |
| Протокол | **Protobuf (nanopb)** | В 5-10 раз компактнее JSON |

#### Критические оптимизации

1. **LTTB (Largest-Triangle-Three-Buckets)**
   ```dart
   // 100,000 точек → 1,000 для графика
   List<Point> downsample(List<Point> data, int threshold);
   ```

2. **Isolates для параллельной обработки**
   - Isolate 1: Чтение данных, парсинг, фильтрация
   - Isolate 2: UI, графики

3. **Protobuf вместо JSON**
   ```protobuf
   message SensorPacket {
     uint32 timestamp_ms = 1;
     float voltage_v = 2;
     float current_a = 3;
     // ...
   }
   ```

### 4.2 OTA-обновления через BLE

```
┌─────────────────────────────────────────┐
│  Процесс обновления:                    │
│                                         │
│  1. Приложение проверяет версию         │
│  2. Скачивает .bin с сервера            │
│  3. Передаёт на датчик (chunked)        │
│  4. ESP32 записывает в OTA-раздел       │
│  5. Перезагрузка с новой прошивкой      │
│                                         │
│  Безопасность: CRC32 + Secure Boot      │
└─────────────────────────────────────────┘
```
