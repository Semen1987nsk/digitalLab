### 5.3 Killer Features (чего нет у конкурентов)

1. **Web UI в датчике (192.168.4.1)**
   - Fallback без приложения
   - Работает в любом браузере
   - Учитель видит показания через телефон

2. **OTA-обновления через BLE**
   - Новые функции без визита сервис-инженера
   - Исправление багов удалённо

3. **Мультиплеер (2+ устройства)**
   - Два ученика — один график
   - Один меряет силу, другой — перемещение
   - Совместная лаборатория

4. **LTTB + Kalman (производительность)**
   - 100,000 точек → 1,000 для графика
   - Работает на Celeron N4000 без лагов

5. **Protobuf вместо JSON**
   - В 5-10 раз меньше трафика
   - Быстрее парсинг

## 8. Выводы

### Главный конкурент: **Releon**

Они контролируют сегмент современных беспроводных решений. Однако у них есть критические слабости:
- Нет Web UI
- Нет OTA
- Вероятно, нет BLE 5.0

### Стратегия победы

1. **Технологическое превосходство** — Web UI, OTA, BLE 5.0, Protobuf
2. **Производительность** — LTTB, Isolates, работа на старых ПК
3. **Astra Linux native** — сертификат от ГК "Астра"
4. **Открытость** — публичный протокол, API для интеграций
5. **Цена** — на 15-20% ниже Releon при лучших характеристиках

### Критерий успеха

> Ваш продукт должен быть **единственным**, который:
> - Работает без приложения (через браузер)
> - Обновляется по воздуху
> - Нативно работает на Astra Linux
> - Позволяет объединять несколько датчиков в один эксперимент
